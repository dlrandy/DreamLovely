# Behavioral Patterns
行为设计模式关注的是对象之间的责任的分配和算法

- 职责链模式(chain of responsibility)
  沿着handler chain来传递requests。一旦接受request，每一个handler
  来决定是处理请求还是将其传递给链里的下一个handler
- 命令模式(command)
  将request变成独立的对象，该对象包含request的所有信息。这种转换
  可以对不同的requests使用不同的方法，延迟或者入队request的执行和
  支持不能做的操作
- 迭代器(Iterator)
  在不需要暴露底层描述的情况下，遍历集合里的每一个元素
- 中介者(Mediator)
  减少对象之间的混乱的依赖。这个模式限制了对象之间的直接通信，强迫对象
  之间通过mediator对象
- 备忘录(memento)
  在不展示实现细节的情况下保存和恢复对象之前的状态
- 观察者(Observer)
  定义注册机制在对象们监听的事件发生的时候来通知对象，
- 状态(state)
  当对象内部状态变化的时候，对象改变自身行为，就好像
  对象改变了它的class
- 策略(Strategy)
  定义一组算法，把他们每一个放到独立的class里，使得对象
  之间可以互换
- 模板方法(template method)
  在父类里定义算法的骨架，但是让子类去覆盖算法特定的步骤，但是
  不改变算法的结构
- 访问者
  将算法和要运行算法的对象进行分离
